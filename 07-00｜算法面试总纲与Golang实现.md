## 一、算法学习地图与现场答题框架

### 1. 后端算法知识图谱是什么？

```text
Complexity / Constraint
  -> Array / Hash / Prefix
  -> Two Pointer / Window / Binary Search
  -> Linked List / Stack / Queue
  -> Tree / Heap / Divide and Conquer
  -> Graph / Union Find / Backtracking
  -> Dynamic Programming / Greedy
  -> Go Implementation / Test / Engineering Problem
```

### 2. 现场答题的标准七步是什么？

复述题意与输入 -> 问清边界 -> 给暴力和瓶颈 -> 识别模板/不变量 -> 说明复杂度 -> 编码 -> 手工用例和边界自测。不要拿到题立刻沉默写代码。

### 3. 什么是算法不变量？

循环/递归每一步都成立的条件，例如滑动窗口内满足约束、二分答案区间始终包含解、链表 prev 指向已反转部分、堆顶是当前最小候选。先说不变量能显著减少 Bug。

### 4. 卡住时怎样推进？

写小样例、画结构、给 O(n²) 暴力，问重复计算在哪里；从数据范围推复杂度；尝试排序、Hash、单调性和状态定义。向面试官同步假设而非一直沉默。

### 5. 资深岗位算法评价什么？

不仅是 AC，还看沟通、边界、代码结构、复杂度、测试和是否过度设计。高级候选人不应因工程经验丰富而忽视基础手写。

### 6. 复习应按题还是按母模板？

先掌握能覆盖多题的母模板，再用代表题和变形验证：反转、遍历、窗口、二分、堆、BFS/DFS、回溯和 DP。最后随机混合训练识别能力。

## 二、复杂度、约束与数据结构选择

### 7. 如何从数据规模反推复杂度？

大致思路：n很小可指数/回溯，中等可 O(n²)，10⁵级通常 O(n log n)/O(n)，更大需线性/数学。实际还受常数、语言和时限影响，面试给合理判断即可。

### 8. 时间复杂度为什么不是实际耗时？

它描述增长阶，忽略常数、Cache、分配和输入分布。两个 O(n) 实现可能差很多；但面试先保证阶正确，再谈工程性能。

### 9. 空间复杂度如何计算？

关注额外空间、递归栈、Hash/Visited、输出是否计入按题意说明。原地算法也可能有 O(log n) 递归栈。

### 10. 均摊复杂度是什么？

少数昂贵操作摊到操作序列，如动态数组扩容、单调栈元素进出一次。它不是平均输入概率，而是序列总成本上界。

### 11. 如何选择 Array、Map、Set、Heap？

连续索引/顺序扫描用数组；存在性/映射用 Hash；动态 TopK/最值用 Heap；有序范围需排序、Tree或二分；选择同时说明空间和是否需要稳定顺序。

### 12. 排序为什么经常是第一步？

排序把无序问题转成单调结构，便于双指针、区间合并、二分和去重，代价 O(n log n)及可能丢原下标。需保留原位置时存 Pair。

### 13. 递归和迭代如何选择？

树/回溯递归表达自然，深度可控时优先清晰；链表长、图深或输入不可信时迭代避免栈风险。Go Goroutine 栈可增长但不是无限。

## 三、数组、哈希、字符串、前缀和与区间

### 14. 两数之和代表什么模板？

“遍历当前元素，查询此前状态”：Map记录值到下标，查 target-x，时间 O(n)、空间 O(n)。先查再放处理同一元素不能重复使用。

### 15. Hash 题的四类映射是什么？

值到位置、值到频次、状态到首次位置、前缀值到出现次数。看到查补数、分组、去重、最长连续和子数组计数优先考虑。

### 16. 前缀和解决什么？

将区间和转为 `prefix[r]-prefix[l]`；配合 Map 查此前前缀解决和为 K 的子数组。二维前缀处理矩形查询。注意初始前缀 0 和次数而非仅存在。

### 17. 区间问题的通用步骤是什么？

按起点/终点排序，维护当前合并区间或结束边界；明确闭开区间和相邻是否合并。会议室/重叠数可用扫描线或 Heap。

### 18. 字符串题在 Go 中有什么特殊边界？

Byte适合 ASCII/协议，Rune适合 Unicode Code Point；`range`索引是字节偏移。字符计数数组比 Map快但只适固定字符集；切片必须落在 UTF-8边界。

### 19. 矩阵模拟如何少写 Bug？

明确行列范围、四边界和每轮收缩；方向数组统一邻接遍历；访问前检查边界。空矩阵和单行/单列必须自测。

### 20. 原地数组修改的常见模式是什么？

读写指针：read扫描，write指向下一个保留位置；适合删除/去重/过滤。每步维持 `[0,write)` 已是最终结果。

## 四、双指针、滑动窗口、二分与单调性

### 21. 左右双指针适合什么？

有序数组求两数、三数、盛水、回文；根据比较结果单调移动一侧。核心是证明移动不会错过解。

### 22. 快慢指针适合什么？

链表中点/环/倒数位置，数组原地读写。要明确定义每个指针速度、距离和相遇条件。

### 23. 滑动窗口的识别条件是什么？

连续子数组/子串，左右边界能单调前进，窗口状态可增量维护。固定窗口与可变窗口分开；有负数时某些“和满足就缩”不再单调。

### 24. 可变窗口模板是什么？

右侧加入并更新状态；当不满足/过度满足时移动左侧并撤销；在正确时机更新答案。最长通常在满足时更新，最短常在收缩过程中更新。

### 25. 二分查找的本质是什么？

在单调谓词上寻找边界，而不只是查数。先定义“第一个满足条件”的区间和不变量，再写循环，可覆盖左右边界、旋转数组和答案二分。

### 26. 推荐的左边界二分模板是什么？

```go
func lowerBound(nums []int, target int) int {
	l, r := 0, len(nums)
	for l < r {
		m := l + (r-l)/2
		if nums[m] >= target { r = m } else { l = m + 1 }
	}
	return l
}
```

区间始终是 `[l,r)`，返回第一个大于等于目标的位置。

### 27. 答案二分怎样识别？

题目求最小可行最大值/最大可行最小值，且给定答案 x 能 O(n)判断可行，谓词随 x 单调。边界来自可能答案范围，而不是数组下标。

### 28. 为什么二分最容易死循环？

区间定义混用、Mid取整和更新不缩小范围。写前决定闭区间或半开区间，每个分支保证长度严格减少，用 0/1/2元素测试。

## 五、链表、栈、队列与单调结构

### 29. 链表题写代码前问什么？

头节点会变吗、需要找位置吗、会断链吗、一个还是多个链表、返回头是谁。头可能变用 Dummy，修改 Next 前保存 Next。

### 30. 反转链表的核心不变量是什么？

`prev` 是已反转区间头，`cur` 是未处理首节点。保存 next -> cur.Next=prev -> 推进。区间反转用 `[head,stop)` 模板复用。

### 31. 环入口如何推导？

快慢相遇后，一个从头、一个从相遇点同速，下一次相遇是入口。面试可简述距离等式，并先保证空指针边界。

### 32. 栈适合什么题？

括号、表达式、DFS迭代、路径、单调结构和撤销。判断需要“最近未匹配/后进先出”时使用。

### 33. 单调栈解决什么？

寻找左/右第一个更大/更小、柱状图、每日温度、接雨水。元素通常进出各一次 O(n)；栈存下标便于距离和取值。

### 34. 单调队列解决什么？

滑动窗口最大/最小：队尾移除不可能成为答案的元素，队头移除过期下标。Deque保持值单调且索引在窗口内。

### 35. LRU 为什么是 HashMap + 双向链表？

Map O(1)定位节点，双链表 O(1)移动和淘汰尾部；Dummy简化边界。每次 Get/Put维护“头部最新、尾部最旧”不变量。

### 36. 合并 K 个有序链表如何选择？

小顶堆每次取最小 O(N log K)，或分治两两归并同阶且常数不同。解释 K、总节点 N 和是否可修改输入。

## 六、二叉树、BST、堆、分治与 TopK

### 37. 树题为什么常用递归定义？

问题可由左右子树同类结果组合。先定义函数返回语义，再写终止、子问题和合并；不要边写边改变返回含义。

### 38. 前中后序分别适合什么？

前序先处理根，适合构建/路径；中序在BST得到有序；后序先得子树结果，适合高度、直径、平衡和删除。层序用BFS处理层级最短距离。

### 39. BST 题的核心性质是什么？

中序严格有序或节点受上下界约束。验证不能只比较直接孩子；重复值规则按题意。查第K小用中序计数。

### 40. 树的后序信息汇总模板是什么？

递归返回父节点需要的信息（高度/贡献/是否平衡），同时用外部变量更新全局最优（直径/路径和）。区分“向父返回”与“全局答案”。

### 41. Heap 的不变量是什么？

完全二叉树且父节点不大于/不小于孩子，只保证堆顶极值，不保证全序。插入上浮、删除顶下沉，均 O(log n)。

### 42. TopK 如何选 Heap、Quickselect 或排序？

全排序 O(n log n)简单；K小/流式用大小K Heap O(n log K)；只需一次第K且允许修改用Quickselect平均O(n)。需要有序结果还要最终排序。

### 43. Go `container/heap` 有什么坑？

实现 Len/Less/Swap/Push/Pop，Push/Pop方法使用指针接收者并操作切片尾；小顶/大顶由Less决定。业务代码封装类型，避免到处断言 `any`。

### 44. 分治和递归有什么区别？

分治将问题拆成相对独立子问题并合并，如归并排序、构树；递归只是实现方式。分治需分析子问题数量、规模和合并成本。

## 七、图、BFS/DFS、拓扑排序、并查集与回溯

### 45. 图的存储如何选择？

稀疏图邻接表 O(V+E)空间，稠密/快速边查询可邻接矩阵 O(V²)。无向边双向添加；节点编号连续可Slice，否则Map。

### 46. BFS 和 DFS 如何选择？

无权最短路和层级扩散用 BFS；连通、路径枚举、拓扑递归和回溯用 DFS。两者都需明确Visited何时标记，BFS通常入队即标记防重复。

### 47. 多源 BFS 是什么？

所有源点同时入队、距离0，逐层扩散，得到离最近源的距离/时间，如腐烂橘子。不是分别从每个源跑BFS。

### 48. 拓扑排序解决什么？

有向无环图依赖顺序；Kahn用入度0队列，处理数少于节点数则有环；DFS用三色检测回边。课程、构建和任务依赖常见。

### 49. 并查集适合什么？

动态合并集合和判断连通。Parent+路径压缩+按秩/大小合并，近似 O(1)；适合无向连通、冗余边和账户合并，不适合直接求具体路径。

### 50. 回溯的核心模板是什么？

路径、选择列表、终止条件；做选择 -> 递归 -> 撤销。组合控制起点，排列用Used，重复元素先排序并在同层去重，剪枝要证明不会漏解。

### 51. 网格搜索常见 Bug 是什么？

行列混淆、Visited恢复错误、边界、重复访问和递归深度。连通遍历Visited不恢复；路径回溯离开时恢复。方向数组统一四邻接。

## 八、动态规划、贪心与状态设计

### 52. 如何识别动态规划？

求最优/计数/可行，存在重叠子问题和最优子结构；能定义有限状态并从较小状态转移。先写递归关系，再决定记忆化或迭代。

### 53. DP 五步是什么？

状态含义、转移、初始化、遍历顺序、返回值。再分析是否能压缩空间。状态定义不清时不要先写二维数组。

### 54. 背包问题如何区分？

0/1每物品一次，容量倒序；完全背包可重复，容量正序。求组合数/排列数时物品和容量循环顺序影响计数语义。

### 55. 子序列和子数组 DP 有什么区别？

子数组要求连续，状态常表示“以i结尾”；子序列可跳过，常按前缀或二维序列定义。最长公共子序列和最大子数组代表两类。

### 56. 什么时候用记忆化搜索？

状态转移自然递归、部分状态不会访问、迭代顺序难时。注意缓存Key、基线和递归深度；能清楚自底向上时迭代通常更省栈。

### 57. 贪心如何证明？

局部选择可交换为某个最优解，或证明选择后剩余仍是同类子问题。只凭“每次选最大看起来对”不够；无法证明时考虑DP。

### 58. DP 空间压缩有哪些风险？

覆盖仍需要的旧状态，遍历方向错误，二维依赖不完整。压缩前画出转移依赖，只在面试时间允许和正确性稳定时做。

## 九、Golang 编码、边界、自测与性能

### 59. Go 算法代码的基本规范是什么？

语义命名、函数短、先处理空输入、避免全局状态、整数溢出意识、注释不变量而非语法。面试中标准库可用就用，但关键结构如Heap按题意实现。

### 60. Slice 和 Map 有哪些算法陷阱？

Append可能扩容、子Slice共享数组、nil/空语义；Map遍历无序、并发不安全、读取零值需comma-ok。回溯结果必须复制当前Path，否则后续修改覆盖答案。

### 61. 如何设计自测用例？

最小/空、单元素、一般、重复、全相同、升降序、无解、边界值和触发核心分支。先口述预期，再跟代码逐步执行。

### 62. 如何防止整数溢出？

中点用 `l+(r-l)/2`，和/乘积按范围用int64；比较乘法可改除法但处理0；Go溢出不会自动异常。题目约束决定类型。

### 63. 递归代码如何避免共享状态污染？

每层明确选择/撤销，返回结果与全局最优分开；多次调用前初始化；并发环境不使用包级变量。追加Path到答案时复制。

### 64. 如何分析 Go 实现常数成本？

Heap分配、Map、String/Rune转换、递归、排序和复制；面试先复杂度正确，工程上线再Benchmark/Profile。不要为零分配写不可读Unsafe算法。

### 65. 面试代码写完如何讲复杂度？

定义n/V/E/K，解释每元素进出次数、排序/Heap/递归状态数；空间包含数据结构和调用栈。避免只报结论不说明来源。

## 十、后端工程手写题、复习路线与最终模板

### 66. 后端高频工程手写题有哪些？

LRU、Worker Pool、令牌桶/滑动窗口、TopK、并发安全Map、生产者消费者、单例、延时任务简化版和一致性哈希。既考数据结构，也考并发和工程边界。

### 67. Worker Pool 手写要包含什么？

有界Worker/Queue、Submit Context、Owner关闭、WaitGroup、Panic/Error、停止接收和优雅退出。若只写无限Goroutine不算生产级。

### 68. 限流器手写要关注什么？

算法状态、时间来源、并发安全、突发、精度和清理。令牌桶适合允许突发，滑动窗口更精确但状态多；测试注入Clock避免Sleep。

### 69. 一致性哈希手写要关注什么？

哈希环有序结构、虚拟节点、顺时针查找和回绕；处理Hash冲突、权重和节点增删。工程上还需复制、迁移和热点，面试代码可明确简化。

### 70. 第一轮复习顺序是什么？

链表反转/快慢 -> 树遍历/后序 -> Hash/前缀 -> 双指针/窗口 -> 二分 -> Heap/TopK -> BFS/DFS -> 回溯 -> DP。对应母模板练习册逐项默写。

### 71. 时间不足时如何取舍？

优先 S+ 题和14类母模板，保证能在20分钟写出并自测；再补动态规划和专项。不要平均刷大量相似题却不会独立复现。

### 72. 掌握一道题的标准是什么？

能识别模板、解释不变量和复杂度、无提示写出、覆盖边界、完成一种变形，并在隔天重新写。只看答案不算掌握。

### 73. 如何用两分钟回答算法题？

> 我先确认输入、返回和边界，用数据规模判断复杂度。先给暴力基线，再指出重复计算或单调性，选择Hash、排序双指针、窗口、二分、Heap、图遍历或DP。编码前定义循环/递归不变量和数据结构含义，写Go代码时处理空值、重复、整数范围和Slice复制。完成后用最小、一般、重复和极端用例走一遍，最后准确说明时间、额外空间和可选方案的权衡。
