## 四、链表题通用思维框架

链表题不是“看到题套一个模板”，而是先判断操作本质，再组合模板。

### 4.1 第一反应：先问 5 个问题

```text
1. 会不会改变链表结构？
2. 头节点会不会变？
3. 需要找某个位置吗？
4. 是一个链表，还是两个/多个链表？
5. 是否要求 O(1) 查询、移动或淘汰？
```

| 问题 | 如果答案是“是” | 优先想到 |
|---|---|---|
| 会改变结构吗？ | 删除、插入、反转、合并 | dummy + 保存 next |
| 头节点会变吗？ | 删除头、反转头、合并新链表 | dummy |
| 要找位置吗？ | 倒数、中点、环入口 | 快慢指针 |
| 两个链表有关联吗？ | 相交、合并、相加 | 路径对齐 / 归并 / 进位模拟 |
| 多个链表吗？ | K 个链表 | 分治归并 / 小顶堆 |
| 要 O(1) 查询和移动吗？ | LRU | HashMap + 双向链表 |

### 4.2 通用解题步骤

```text
Step 1：判断是否需要 dummy
Step 2：判断是否需要定位节点
Step 3：判断是否需要断链/改指针
Step 4：判断是否需要多个指针协作
Step 5：写代码前先明确每个指针含义
Step 6：最后检查头节点、nil、断链、返回值
```

### 4.3 Step 1：先判断要不要 dummy

只要有下面任意一种情况，优先用 dummy：

```text
可能删除头节点
可能返回新的头节点
需要拼接新链表
需要反转局部区间
需要统一处理边界
```

典型题：

```text
19 删除倒数第 N 个节点
21 合并两个有序链表
2 两数相加
92 反转链表 II
25 K 个一组翻转链表
```

一句话：

> 链表题只要“头节点可能变”，先上 dummy。

基础写法：

```go
dummy := &ListNode{Next: head}
// ...
return dummy.Next
```

构造新链表：

```go
dummy := &ListNode{}
tail := dummy
// tail.Next = node
// tail = tail.Next
return dummy.Next
```

### 4.4 Step 2：再判断是不是“找位置”

链表不能随机访问，所以很多题本质是“找位置”。

| 要找什么 | 模板 |
|---|---|
| 倒数第 N 个 | 快慢指针，fast 先走 N 步 |
| 中点 | 快慢指针，fast 走 2 步，slow 走 1 步 |
| 是否有环 | 快慢指针相遇 |
| 环入口 | 相遇后 head 和 meet 一起走 |
| 反转区间前驱 | dummy + 循环走 left-1 步 |
| K 个一组尾节点 | 从 groupPrev 往后数 K 个 |

关键判断：

```text
我要找到的是目标节点，还是目标节点的前一个节点？
```

删除题通常找“目标节点的前一个节点”；反转题通常找“区间前一个节点 + 区间后一个节点”。

### 4.5 Step 3：如果要改指针，先保存 next

只要看到反转、删除、插入、断开、拼接，先记一句：

```text
改 cur.Next 前，先保存 next。
```

反转链表四步口诀：

```text
存 next
改指向
prev 前进
cur 前进
```

对应代码：

```go
next := cur.Next
cur.Next = prev
prev = cur
cur = next
```

### 4.6 Step 4：判断一个链表、两个链表、还是多个链表

一个链表常见任务：

```text
反转
删除
找环
找中点
排序
```

优先模板：

```text
dummy
快慢指针
三指针反转
归并排序
```

例如 148 排序链表不是单模板，而是组合：

```text
快慢指针找中点 + 断链 + 递归排序 + mergeTwoLists
```

两个链表常见任务：

| 题目 | 本质 | 模板 |
|---|---|---|
| 21 合并两个有序链表 | 两个有序序列归并 | dummy + 双指针 |
| 160 相交链表 | 长度不同但找同一节点 | 路径对齐 |
| 2 两数相加 | 两个链表同步模拟加法 | dummy + carry |

看到两个链表时，先问：

```text
它们是要比较顺序？
还是要对齐路径？
还是要逐位模拟？
```

多个链表常见是 23 合并 K 个升序链表：

```text
分治归并 / 小顶堆
```

如果用分治，本质还是复用 `mergeTwoLists`。

### 4.7 Step 5：写代码前定义指针语义

写代码前先说清楚这些指针：

```text
dummy：虚拟头节点
prev：当前处理位置的前一个节点
cur：当前节点
next：保存 cur.Next，防止断链
fast：快指针
slow：慢指针
groupPrev：当前 K 组前一个节点
kth：当前 K 组最后一个节点
groupNext：当前 K 组后一个节点
tail：构造新链表的尾指针
```

例如 K 个一组翻转：

```text
groupPrev：这一组前面的节点
kth：这一组最后一个节点
groupNext：下一组的开头
```

### 4.8 链表题的思想分类

| 思想 | 代表模板 | 解决问题 | 关键词 | 代表题 |
|---|---|---|---|---|
| 哨兵思想 | dummy | 头节点特殊处理、删除头、构造新链表 | 删除、插入、合并、新链表、局部反转 | 19、21、2、92、25 |
| 双指针定位思想 | fast / slow | 倒数位置、中点、环、环入口 | 倒数、中点、是否有环、入口 | 19、141、142、148 |
| 局部结构改造思想 | 三指针反转、区间反转、K 组反转 | 改变链表方向 | 反转、翻转、每 K 个、left 到 right | 206、92、25 |
| 归并思想 | mergeTwoLists、分治 merge | 合并有序链表、排序链表 | 有序、合并、排序、K 个 | 21、23、148 |

### 4.9 链表题快速决策树

```text

    是 -> 三指针 / 区间反转 / K 组反转
    否 -> 继续

题目要删除/插入/构造新链表吗？
    是 -> dummy
    否 -> 继续

题目要找倒数、中点、环吗？
    是 -> 快慢指针
    否 -> 继续

题目有两个链表吗？
    是 ->
        有序合并？ -> mergeTwoLists
        找相交？ -> 路径对齐
        数字相加？ -> carry 模拟
    否 -> 继续

题目有 K 个链表吗？
    是 -> 分治归并 / 小顶堆
    否 -> 继续

题目要求 O(1) 查询和淘汰吗？

```

### 4.11 写链表代码前的检查清单

```text
1. 我要不要 dummy？
2. 我要返回 head 还是 dummy.Next？
3. 我要找目标节点，还是目标节点前一个节点？
4. 有没有可能删除/反转头节点？
5. 改 Next 前有没有保存 next？
6. 是否需要断链？
7. 循环条件是 cur != nil，还是 fast != nil && fast.Next != nil？
8. 最后有没有接回剩余链表？
```

尤其注意：

```text
删除题：找前一个节点
反转题：保存 next
合并题：移动 tail/cur
快慢题：防止 fast.Next 空指针
K 组反转：不足 K 个不要反转
```

```text
第一组：dummy + 基础指针
```

## 母模板 1：链表区间反转

### 识别关键词

```text
反转链表
反转 left 到 right
每 K 个一组反转
局部反转
```

### 核心不变量

```text
previous：已经反转部分的头
current：当前待处理节点
next：提前保存，防止断链
stop：本次反转区间的后一个节点
```

### 通用模板：反转 [head, stop)

```go
func reverseRange(head, stop *ListNode) *ListNode {
	previous := stop
	current := head

	for current != stop {
		next := current.Next
		current.Next = previous
		previous = current
		current = next
	}

	return previous
}
```

### 如何套题

#### 206：反转整个链表

```go
func reverseList(head *ListNode) *ListNode {
	return reverseRange(head, nil)
}
```

#### 92：反转局部区间

```text
1. dummy 指向 head
2. 找到 left 前一个节点 before
3. 找到 right 后一个节点 stop
4. before.Next = reverseRange(区间头, stop)
```

#### 25：K 个一组反转

```text
1. 从 groupPrev 向后找第 K 个节点
2. 不足 K 个直接结束
3. 保存 groupNext
4. reverseRange(groupHead, groupNext)
5. 旧 groupHead 变成新尾部
```

### 覆盖题目

- 206. 反转链表
- 92. 反转链表 II
- 25. K 个一组翻转链表

## 母模板 2：二叉树后序信息汇总

### 识别关键词

```text
左右子树给当前节点返回信息
高度
是否平衡
直径
最大路径
最近公共祖先
```

### 核心思考

每次先问四个问题：

```text
1. 左子树返回什么？
2. 右子树返回什么？
3. 当前节点如何合并？
4. 向父节点返回什么？
```

### 通用模板

```go
func solveTree(root *TreeNode) int {
	answer := 0

	var dfs func(node *TreeNode) int

	dfs = func(node *TreeNode) int {
		if node == nil {
			return 0
		}

		left := dfs(node.Left)
		right := dfs(node.Right)

		// 在这里使用 left、right 更新全局答案
		answer = max(answer, left+right)

		// 向父节点只能返回一条方向的信息
		return max(left, right) + 1
	}

	dfs(root)
	return answer
}
```

### 各题只改两个地方

| 题目 | 全局答案怎么更新 | 向父节点返回什么 |
|---|---|---|
| 104 最大深度 | 不需要全局答案 | `max(left,right)+1` |
| 110 平衡二叉树 | 高度差大于 1 返回 -1 | 子树高度 |
| 543 二叉树直径 | `left+right` | `max(left,right)+1` |
| 124 最大路径和 | `node.Val+left+right` | `node.Val+max(left,right)` |
| 236 最近公共祖先 | 左右都非空返回当前节点 | 找到的目标节点 |

### 覆盖题目

- 104. 二叉树的最大深度
- 110. 平衡二叉树
- 543. 二叉树的直径
- 124. 二叉树中的最大路径和
- 236. 二叉树的最近公共祖先

## 母模板 3：二叉树遍历、BST 与层序

这类题不是计算复杂状态，而是“以正确顺序访问节点”。

### 3.1 DFS 遍历骨架

```go
func dfs(root *TreeNode) {
	if root == nil {
		return
	}

	// 前序位置
	dfs(root.Left)

	// 中序位置
	dfs(root.Right)

	// 后序位置
}
```

### 3.2 中序迭代模板

```go
func inorderTraversal(root *TreeNode) []int {
	answer := make([]int, 0)
	stack := make([]*TreeNode, 0)
	current := root

	for current != nil || len(stack) > 0 {
		for current != nil {
			stack = append(stack, current)
			current = current.Left
		}

		current = stack[len(stack)-1]
		stack = stack[:len(stack)-1]

		answer = append(answer, current.Val)
		current = current.Right
	}

	return answer
}
```

### 3.3 层序 BFS 模板

```go
func levelOrder(root *TreeNode) [][]int {
	if root == nil {
		return nil
	}

	queue := []*TreeNode{root}
	answer := make([][]int, 0)

	for len(queue) > 0 {
		size := len(queue)
		level := make([]int, 0, size)

		for i := 0; i < size; i++ {
			node := queue[0]
			queue = queue[1:]

			level = append(level, node.Val)

			if node.Left != nil {
				queue = append(queue, node.Left)
			}
			if node.Right != nil {
				queue = append(queue, node.Right)
			}
		}

		answer = append(answer, level)
	}

	return answer
}
```

### 如何套题

| 题目 | 使用方式 |
|---|---|
| 94 中序遍历 | 直接使用中序模板 |
| 98 验证 BST | 中序结果必须严格递增 |
| 230 BST 第 K 小 | 中序访问到第 K 个节点 |
| 101 对称二叉树 | 同时比较左树左侧与右树右侧 |
| 226 翻转二叉树 | 遍历每个节点并交换左右孩子 |
| 102 层序遍历 | 直接使用 BFS |
| 199 右视图 | 保存每层最后一个节点 |

### 覆盖题目

```go
		fast = fast.Next
```

```go
func hasCycle(head *ListNode) bool {

		}
	}

	return false
}
```

```go
func slidingWindow(s string) int {
	count := make(map[byte]int)
	left := 0
	answer := 0

		count[in]++
```

```go
func lowerBound(nums []int, target int) int {
	left, right := 0, len(nums)

	for left < right {
		mid := left + (right-left)/2

		if nums[mid] >= target {
```

```go
func binarySearch(nums []int, target int) int {
	left, right := 0, len(nums)-1

	for left <= right {
		mid := left + (right-left)/2

		if nums[mid] == target {
			return mid
		}

		if nums[mid] < target {
			left = mid + 1
		} else {
			right = mid - 1
		}
	}

	return -1
}
```

### Go 小顶堆定义

```go
type IntMinHeap []int

func (h IntMinHeap) Len() int           { return len(h) }
func (h IntMinHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h IntMinHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

func (h *IntMinHeap) Push(value any) {
	*h = append(*h, value.(int))
}

func (h *IntMinHeap) Pop() any {
	old := *h
	n := len(old)
	value := old[n-1]
	*h = old[:n-1]
	return value
}
```

### 前 K 大通用模板

```go
func topK(nums []int, k int) []int {
	h := &IntMinHeap{}
	heap.Init(h)

	for _, value := range nums {
		heap.Push(h, value)

		if h.Len() > k {
			heap.Pop(h)
		}
	}

	return append([]int(nil), (*h)...)
}
```

```go
func heapSort(nums []int) {
	n := len(nums)

	for i := n/2 - 1; i >= 0; i-- {
		siftDown(nums, i, n)
	}

	for end := n - 1; end > 0; end-- {
		nums[0], nums[end] = nums[end], nums[0]
		siftDown(nums, 0, end)
	}
}

func siftDown(nums []int, root, size int) {
	for {
		left := root*2 + 1
		if left >= size {
			return
		}

		largest := left
		right := left + 1

		if right < size && nums[right] > nums[left] {
			largest = right
		}

		if nums[root] >= nums[largest] {
			return
		}

		nums[root], nums[largest] = nums[largest], nums[root]
		root = largest
	}
}
```
