# 后端算法面试 70 题高频分类清单

> 适用目标：字节跳动、腾讯等互联网公司后端岗位现场算法面试。  
> 选题原则：优先考虑现场面试中容易出现的题型、常见原题和高频变形，以及解法模板的覆盖价值。

## 优先级说明

- **S+：必须默写**  
  建议在 15～20 分钟内独立完成，能说明复杂度和关键边界，并处理常见变形。

- **S：必须掌握**  
  能够独立写出主体代码，理解模板，并能完成简单变形。

- **A：补齐模板**  
  至少能够快速识别题型，讲出解法并写出核心代码。

**链表题模板组合速查：**

链表题先不要急着写代码，先按这个顺序判断：

```text
是否头节点会变？        -> dummy
是否要找倒数/中点/环？  -> 快慢指针
是否要改 next 指针？    -> 先保存 next
是否两个链表长度不同？  -> 路径对齐
是否有序链表合并/排序？ -> 归并
是否逐位相加？          -> dummy + carry
是否 O(1) 查询和淘汰？  -> HashMap + 双向链表
```

# 十三、最终结论

这 70 题的目标不是保证原题直接命中，而是覆盖后端现场算法面试中大多数常见题型和解法模板。

真正有效的完成标准是：

```text
看到题目后能够快速识别所属模板
5 分钟内讲清楚解题思路
15～25 分钟写出完整代码
主动说明时间和空间复杂度
覆盖空输入、越界、重复元素等边界
题目稍作变化后仍然能够调整代码
```

当你能够把多道题归纳为同一个模板时，才算真正形成面试能力。例如：

```text
206、92、25 → 链表反转模板
104、110、543、124 → 后序 DFS 模板
3、76 → 滑动窗口模板
34、33、153、162 → 二分查找模板
200、79、994 → 网格搜索模板
46、78、39 → 回溯模板
```
